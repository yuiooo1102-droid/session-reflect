目标漂移检测——对比用户声明的目标和实际行为，发现偏差。

## 执行步骤

1. 同步新 session：
```bash
python3 ~/coding/session-reflect/extract_sessions.py sync
```

2. 读取配置获取 vault 路径：
```bash
cat ~/.config/session-reflect/config.json
```

3. 从 Vault 中寻找用户声明的目标、计划、优先级（用户自己写的笔记 + 对话记录，最近 7 天）

4. 统计实际对话中的时间分配（按项目/主题）

5. 生成漂移报告，写入 `{vault}/自我观察/{YYYY-MM-DD}-drift.md`：

```markdown
---
date: {YYYY-MM-DD}
type: drift
period: {起始日期} ~ {结束日期}
---

# 目标漂移检测 — {date}

## 声明的目标
{从笔记/对话中提取的用户目标}

## 实际时间分配
{按项目/主题统计对话占比}

## 漂移信号
{目标和行为之间的偏差，附证据}

## 沉默区域
{声明重要但最近完全没出现的事项}

## 建议
{1-2 条建议：是该回归原目标，还是承认优先级已经变了并更新目标？}
```

## 原则
- 证据驱动，每个偏差都要有具体内容佐证
- 对话记录反映行为，用户笔记反映意图
- 不评判，只指出差异并给出务实建议
- 中文输出
